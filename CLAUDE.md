# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

A personal profile/knowledge website ("In-progress Engineer") built with [Docusaurus 3](https://docusaurus.io/) and TypeScript. The bulk of the content is technical learning notes written as `.mdx` files under `docs/`, organized into topic categories (ai, algorithms, computers, networking, etc.). The site is deployed automatically from `main` via Vercel.

## Commands

- `npm install` - install dependencies (Node >=22.21.0, NPM >=11.7.0 required per `package.json` engines)
- `npm start` - local dev server with live reload
- `npm run build` - production build into `build/` (note: the README's `npm build` is wrong; use `npm run build`)
- `npm run serve` - serve a built site
- `npm run typecheck` / `tsc --noEmit` - TypeScript check (no emit)

Linting and validation are orchestrated through [Task](https://taskfile.dev) (`Taskfile.yaml`), not NPM scripts:

- `task build` - the full clean-build gate; runs `format-lint`, `vale-lint`, and `check-theme-image`. **This must pass before any PR (see AGENTS.md).**
- `task format-lint` - `prettier --write` + `eslint . --max-warnings=0` + `knip` (unused deps/exports)
- `task vale-lint` - prose/spell linting via the `jdkato/vale` Docker image
- `task check-theme-image` - runs `scripts/check-theme-image-usage.mjs`

CI (`.github/workflows/ci.yml`) additionally runs `tsc --noEmit`, `prettier --check`, `npm run build`, and `package-lock-utd` on every PR to `main`.

## Hard rules (from AGENTS.md)

- Every change goes through a PR; **an agent must not approve the PR** - only tell the user it was opened.
- PRs are **rebased and merged** (use that merge option in GitHub).
- `task build` must succeed before opening a PR.
- **Don't install any new NPM packages** at project or global level that aren't part of the package.json, and don't install system binaries/libraries. If a tool is truly needed, use a temporary location only. (`task` itself may be installed if missing.)
- Dependency-upgrade PRs must pass `npx --yes package-lock-utd@1.1.3`.
- Don't commit any changes directly. Only make changes and STOP.

## Architecture & conventions

- **Content lives in `docs/<category>/*.mdx`.** Sidebar categories in `sidebars.ts` use Docusaurus `autogenerated` against each directory name, so adding a new `.mdx` to an existing category folder automatically lists it - no sidebar edit needed. Adding a _new_ category requires a new entry in `sidebars.ts`.
- **Math and diagrams** are enabled site-wide: `remark-math` + `rehype-katex` for LaTeX, and `@docusaurus/theme-mermaid` for Mermaid diagrams.
- **Excalidraw images must use the `ThemeImage` component, not markdown image syntax.** `ThemeImage` (`src/components/ThemeImage/`) applies a CSS `invert` filter so diagrams adapt to dark mode. `check-theme-image-usage.mjs` enforces this by failing the build if any `![...](....excalidraw.png)` markdown image is found. Use:

    ```mdx
    import { ThemeImage } from '../../src/components/ThemeImage';
    <ThemeImage src="/img/xxx.excalidraw.png" alt="Description" />
    ```

- **Styling** uses Tailwind via a custom Docusaurus plugin in `plugins/tailwind-plugin.ts` (config in `tailwind.config.ts`); global CSS overrides live in `src/css/`.
- **Custom React pages/components** are in `src/pages/` and `src/components/` (biography, talks, topics). `knip` checks for unused exports - keep `package.json`'s `knip.ignore`/`ignoreDependencies` in sync when adding intentionally unreferenced code.
- The site config (navbar, plugins, presets, deployment) is in `docusaurus.config.ts`.

## Vale prose linting

- Vale runs in Docker (no local install needed). To add accepted spellings/terms, edit `.github/vale/styles/config/vocabularies/website/accept.txt`. Run locally with `task vale-lint`.
- Don't add anything to accept.txt without approval.

## Readability (Flesch-Kincaid)

The Vale config enables the **Readability** style's `Readability.FleschKincaid` rule at error level (vendored under `.github/vale/styles/Readability/`, tracked via a `.gitignore` negation since it can't be fetched by `vale sync`). Docs must read at grade level <= 8. Keep sentences short and words plain; favor contractions (the Microsoft style enforces them).
