# Device Drivers

They run in kernel space. They let userspace apps work with the hardware device.

1. It must implement the standard VFS methods, such as **open(), read(), write(), ioctl()**.
2. These methods read the initial config from the device registers.
3. They write data to buffers and read data from buffers.
4. They update the device registers, such as the volume for audio devices.

:::tip high level explanation
This is a high level idea of device drivers.
You still need to know the role of each component.
:::

:::warning controller driver
The controller driver is usually not called out on its own.
It's part of the device driver.
The controller driver talks to the device through the device controller chip on the motherboard.

Example - say a USB audio device is plugged in. The device driver uses the
USB controller driver. It then controls the device over the USB bus.
:::
